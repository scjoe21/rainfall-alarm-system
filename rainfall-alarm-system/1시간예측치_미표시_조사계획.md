# 1시간 예측치 지도 미표시 원인 조사 계획

## 현상
- **1시간 실측치**, **15분 실측치**: 지도에 정상 표시
- **1시간 예측치**: 지도에 표시되지 않음 (예: 경상북도 영덕군)

---

## 1. 데이터 흐름 요약

| 데이터 | 소스 | 저장소 | 지도 API |
|--------|------|--------|----------|
| 15분 실측치 | AWS APIHUB (RN-15m) / 공공 API | `aws_rainfall` → `rainfall_realtime` | `getLatestRainfallByDistrict` |
| 1시간 실측치 | AWS APIHUB (RN-60m) / RN1 | `aws_rainfall` → `rainfall_realtime` | 동일 |
| 1시간 예측치 | VSRT (nph-dfs_vsrt_grd) / 초단기예보 | `aws_rainfall` → `rainfall_forecast` | 동일 |

---

## 2. 의심 원인 (우선순위)

### 2-1. AWS 갱신 범위 제한 (가능성 높음)

**위치**: `scheduler.js` → `runAwsRefresh()`

- `getAwsStationsWithRainfall()`: APIHUB에서 **강수가 있는** AWS만 반환
- `getAwsStationsForFallback()`: APIHUB 실패 시 **최대 35개** 관측소만 처리
- **결과**: 영덕군 인근 AWS가 이 목록에 없으면 `aws_rainfall`에 저장되지 않음
- **영향**: 해당 지역 `weather_station`은 sync 시 가장 가까운 AWS를 찾지 못하거나, 예측치가 없는 AWS에만 매칭됨

**확인 방법**:
- `aws-stations-fallback.json`에 영덕군 인근 AWS(예: 영덕, 포항, 울진 등) 포함 여부 확인
- APIHUB 사용 시 `getAwsStationsWithRainfall` 반환 목록에 해당 지역 AWS 포함 여부 확인

---

### 2-2. syncAwsToRainfallRealtime 필터

**위치**: `alarmService.js` 255~260행

```javascript
const awsRows = db.prepare(`
  SELECT stn_id, lat, lon, rainfall_15min, rainfall_1hour, forecast_hourly
  FROM aws_rainfall
  WHERE updated_at >= datetime('now', '-60 minutes')
    AND (rainfall_15min IS NOT NULL OR rainfall_1hour IS NOT NULL)
`).all();
```

- `rainfall_15min`과 `rainfall_1hour`가 **둘 다 NULL**이면 해당 AWS는 제외
- **예측치만 있는** AWS는 sync 대상에서 빠짐
- 실측치는 있는데 예측치가 0으로만 나오는 경우, 이 필터 자체보다는 2-1이 더 유력

---

### 2-3. VSRT/예측 API 실패 또는 0 반환

**위치**: `kmaAPI.js` → `getVsrtForecastHourly()`

- `KMA_APIHUB_KEY` 미설정 시 `getForecast45min`(초단기예보)으로 폴백
- VSRT 격자 누락 시 해당 격자에 대해 0 반환
- API 오류 시 0 반환 → `forecast_hourly`가 0으로 저장되어 지도에 "0.0mm/hr"로 표시

**확인 방법**:
- `/api/status?test=2` 호출로 예측 API 동작 여부 확인
- 서버 로그에서 `[API] VSRT` 관련 메시지 확인

---

### 2-4. AWS–weather_station 거리 제한

**위치**: `alarmService.js` 281행

```javascript
let minDist = 2.0; // ~140km 이내
```

- `weather_station`에서 가장 가까운 AWS가 140km를 넘으면 매칭 실패
- 영덕군(동해안)은 AWS 밀도가 낮을 수 있어, 일부 읍면동에서 매칭 실패 가능

---

### 2-5. rainfall_forecast 시간 조건

**위치**: `alarmService.js` 328~332행

```javascript
AND base_time >= datetime('now', '-60 minutes')
```

- 60분 이내 `base_time`을 가진 예측 데이터만 사용
- sync가 60분 이상 지연되면 해당 구역 예측치가 0으로 처리됨

## 3. 조사 절차 (플랜)

### Step 1: API 응답 확인

1. 영덕군 `districtId` 조회 (예: 1516)
2. `GET /api/rainfall/district/1516` 호출
3. 응답에서 `forecast_hourly` 값 확인
   - 전부 0 → 예측 API 또는 sync 문제
   - null/누락 → JOIN 또는 sync 로직 문제

### Step 2: DB 직접 확인

```sql
-- 영덕군 emd 및 weather_station
SELECT e.id, e.code, e.name, ws.id as ws_id, ws.stn_id, ws.lat, ws.lon
FROM emds e
LEFT JOIN weather_stations ws ON ws.emd_id = e.id
WHERE e.district_id = 1516;

-- 최근 aws_rainfall (영덕 인근 위경도 범위)
SELECT stn_id, name, lat, lon, rainfall_15min, rainfall_1hour, forecast_hourly, updated_at
FROM aws_rainfall
WHERE updated_at >= datetime('now', '-60 minutes')
  AND lat BETWEEN 36.0 AND 37.0 AND lon BETWEEN 129.0 AND 130.0;

-- rainfall_forecast (해당 station_id)
SELECT rf.*, ws.name
FROM rainfall_forecast rf
JOIN weather_stations ws ON rf.station_id = ws.id
JOIN emds e ON ws.emd_id = e.id
WHERE e.district_id = 1516
  AND rf.base_time >= datetime('now', '-60 minutes');
```

### Step 3: 예측 API 상태 확인

- `GET /api/status?test=2` 호출
- VSRT/초단기예보 정상 여부 및 에러 메시지 확인

### Step 4: 로그 확인

- `[AWS10m]` 로그: 갱신된 AWS 개수
- `[API] VSRT` 로그: 격자 로드, 캐시 미스, 폴백 여부
- sync 관련: `rainfall_realtime 동기화 N개` 메시지

## 4. 수정 제안 (원인별)

| 원인 | 수정 방향 |
